# Pneumonia AI Screening

Study-driven full-stack project for pneumonia screening from chest X-ray images.

This repository combines:

- a training workflow based on a chest X-ray study notebook,
- a Flask inference API for prediction and Grad-CAM visualization,
- a Next.js frontend for upload, analysis, and result review,
- an optional Hugging Face Space demo version.

The project is designed as an educational AI prototype and not as a clinical diagnosis system.

## Project Overview

The goal of this project is to classify frontal chest X-ray images into two classes:

- `0 -> Pneumonia`
- `1 -> Normal`

The study pipeline follows a simple and practical approach:

1. load and inspect the Kaggle chest X-ray dataset,
2. preprocess grayscale radiographs into a unified input size,
3. address class imbalance with a GAN that generates additional `Normal` images,
4. train a CNN classifier,
5. search for the best decision threshold on test predictions,
6. export the trained model for web inference.

## Study Summary

The README below is based on the study workflow used for the project:

- Dataset: `paultimothymooney/chest-xray-pneumonia`
- Image format: grayscale chest X-rays
- Input size for CNN: `150 x 150`
- GAN image size: `128 x 128`
- Minority class: `Normal`
- Training strategy: CNN + synthetic `Normal` images generated by a GAN

### Data Split Used in the Study

After the real train/validation split, the study reported:

- Train: `4172` images
  - `3099` Pneumonia
  - `1073` Normal
- Validation: `1044` images
  - `776` Pneumonia
  - `268` Normal
- Test: `624` images
  - `390` Pneumonia
  - `234` Normal

### GAN Rebalancing

To reduce class imbalance, the study trained a GAN on the minority class (`Normal`) and injected synthetic images back into the training set.

Reported values from the study:

- Minority class before GAN: `1073` Normal images
- Majority class before GAN: `3099` Pneumonia images
- Synthetic `Normal` images generated and retained: `429`
- Rebalanced train set: `4601` images
  - `3099` Pneumonia
  - `1502` Normal

This keeps the original training data while improving minority-class representation.

## Reported Results

The latest study notebook reported the following final test performance:

| Metric | Value |
| --- | ---: |
| Accuracy | `94.07%` |
| Balanced Accuracy | `0.9261` |
| Macro F1 | `0.9353` |
| Weighted F1 | `0.9400` |
| AUC | `0.9622` |
| Best threshold | `0.41` |
| Global score | `0.9355` |

Per-class summary from the reported test results:

- Pneumonia:
  - Precision: `0.93`
  - Recall: `0.98`
  - F1-score: `0.95`
- Normal:
  - Precision: `0.97`
  - Recall: `0.87`
  - F1-score: `0.92`

## Web Application

The web application turns the study model into an interactive screening workflow.

### Frontend

- Framework: `Next.js 14`
- Role: upload X-ray images, display prediction results, request Grad-CAM overlays

### Backend

- Framework: `Flask`
- Inference stack: `TensorFlow / Keras`
- Main endpoints:
  - `GET /health`
  - `POST /predict`
  - `POST /gradcam`

### Backend Behavior

The API:

- decodes the uploaded image,
- validates that it looks like a chest X-ray,
- converts it to grayscale,
- resizes it to `150 x 150`,
- normalizes it to `[0, 1]`,
- predicts the probability of class `1` (`Normal`),
- returns the final class label using a configurable threshold.

Grad-CAM is available to generate an overlay showing the model focus region.

## Quick Start

### 1. Run the Backend

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
copy .env.example .env
python app.py
```

Backend default URL:

```text
http://127.0.0.1:5000
```

### 2. Run the Frontend

```powershell
cd frontend
copy .env.local.example .env.local
npm install
npm run dev
```

Frontend default URL:

```text
http://localhost:3000
```

## Important Threshold Note

The latest study reported a best test threshold of `0.41`.

The backend environment example currently defaults to:

```text
PREDICTION_THRESHOLD=0.45
```

If you want the web app to match the reported study threshold exactly, update `backend/.env` to:

```text
PREDICTION_THRESHOLD=0.41
```
